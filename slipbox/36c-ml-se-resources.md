## Online
- [[85_turing_way]]
- [[@serban_se-ml_2021|SE-ML]]: this is a curated list of best practices in the development of production ML applications. The practices are collected from the real experience of teams of developers around the world and are grouped into areas such as Data, Training, Coding, Deployment, Team, etc.
- [[@sculley_hidden_2015|Hidden Technical Debt in Machine Learning Systems]]: a NIPS paper by Google employees that links the world of ML development to that of software engineering. Specifically, it refers to concepts coming from the world of refactoring, such as code smells, anti-patterns, and more.
- the Software Engineering for ML list in [[143_github_lists]] contains a lot of useful resources, especially as regards data management, model training and tools.
- https://git-lfs.github.com/: this could be a useful tool when using large datasets in a project such as ISI's CMV.
