# Maintaining open source software
*or, the art of growing a community.*

In the usual lifecycle of open source software projects, when the project reaches a good degree of maturity (at the very minimum, it can be used by others), a community starts to gather around the project. Some roles usually appear then:
- the *contributors* are users who, by writing code or in any other way, help the project growing
- the *maintainers* are people who overview the process of writing code and take the project-wide decision about the future of the project itself.

It is very important to set some general rules about how it is expected that the contributions are done by the community, and also what kind of behaviour is generally accepted. There exist a standard set of behaviour guidelines called Contributor Covenant[^cc] which is adopted by over 40.000 open source projects.


[^cc]: https://www.contributor-covenant.org/
